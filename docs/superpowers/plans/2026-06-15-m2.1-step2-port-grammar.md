# M2.1 Step 2 — Port-aware policy grammar + Rego Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the egress port loophole — an allow-listed host currently authorizes *every* TCP port; after this change a bare host means web ports (80/443) only, and any other port must be explicitly listed in `policy.yaml`.

**Architecture:** The dialed port is *already* carried end-to-end (`FlowDesc.port` → `input.port` from both the tier-1 MITM `PolicyAdapter` and the tier-2 `decide_tier2` path). So this is a **data-contract + Rego change only, no datapath plumbing**: (1) the per-sandbox policy file grammar gains an optional `ports` list per host (`AllowEntry`), (2) the Rego data document becomes a `host → [ports]` map instead of a `host` list, and (3) `egress.rego` gains a port predicate. The embedded default data maps every shipped host to `[80, 443]`, so existing port-443 traffic is unaffected and existing string-list policies keep parsing — they just now mean "80/443 only" (the intended tightening).

**Tech Stack:** Rust (serde / serde_yaml / serde_json), `regorus` (pure-Rust OPA/Rego, `rego.v1`).

**Reference spec:** [`docs/superpowers/specs/2026-06-15-m2.1-interactive-firewall-design.md`](../specs/2026-06-15-m2.1-interactive-firewall-design.md) § "Step 2 — Port-aware policy grammar + Rego" (lines 182–286).

## The shared data-contract (why Task 1 is one atomic task)

Three artifacts encode the **same** Rego data shape and MUST change together in one commit, or intermediate states deny everything:

- `egress_data.json` — the embedded default data document
- `EgressPolicyConfig::to_rego_data_json` — the per-`--policy` data document
- `egress.rego` — the consumer

Old shape → new shape:

```jsonc
// OLD
{ "global_domains": ["api.anthropic.com", ...],
  "sandbox_domains": { "web": ["api.stripe.com"] } }

// NEW  (host → list-of-allowed-ports)
{ "global_domains": { "api.anthropic.com": [80, 443], ... },
  "sandbox_ports":  { "web": { "api.stripe.com": [80, 443] } } }
```

Key rename: `sandbox_domains` → `sandbox_ports`. Both maps are `host → [u16]`.

- [ ] **Step 3: Introduce `AllowEntry` and switch the field type**

In `crates/izba-core/src/daemon/egress/config.rs`, change the `use serde::Deserialize;` import to `use serde::{Deserialize, Serialize};`, then replace the `EgressPolicyConfig` struct and add `AllowEntry` above it:

```rust
/// One entry in a sandbox's egress allow-list: either a bare host (which
/// authorizes the default web ports) or a host scoped to an explicit port set.
///
/// `#[serde(untagged)]` keeps every existing `allow: [<string>...]` file parsing
/// unchanged — a YAML string deserializes to `Host`, a `{host, ports}` map to
/// `Scoped`. Variant order matters: `Host` is tried first.
#[derive(Debug, Clone, PartialEq, Eq, Deserialize, Serialize)]
#[serde(untagged)]
pub enum AllowEntry {
    /// Bare host → implicit web ports [80, 443].
    Host(String),
    /// Host scoped to an explicit port set (REPLACES the default web ports).
    Scoped { host: String, ports: Vec<u16> },
}

impl AllowEntry {
    /// Ports a bare host authorizes when no explicit set is given.
    pub const DEFAULT_PORTS: [u16; 2] = [80, 443];

    /// The host this entry names.
    pub fn host(&self) -> &str {
        match self {
            AllowEntry::Host(h) => h,
            AllowEntry::Scoped { host, .. } => host,
        }
    }

    /// The ports this entry authorizes: `[80, 443]` for a bare host, else the
    /// explicit set (which REPLACES — not extends — the default).
    pub fn ports(&self) -> Vec<u16> {
        match self {
            AllowEntry::Host(_) => AllowEntry::DEFAULT_PORTS.to_vec(),
            AllowEntry::Scoped { ports, .. } => ports.clone(),
        }
    }
}

/// A sandbox's egress allow-list, parsed from its `--policy` YAML.
#[derive(Debug, Clone, Default, PartialEq, Eq, Deserialize, Serialize)]
pub struct EgressPolicyConfig {
    /// Destinations this sandbox may reach (HTTP host for tier-1, DNS-snoop
    /// FQDN for tier-2). A bare host means web ports (80/443) only; a
    /// `{host, ports}` entry names the exact ports allowed for that host.
    #[serde(default)]
    pub allow: Vec<AllowEntry>,
}
```

- [ ] **Step 4: Rewrite `to_rego_data_json` to emit the host→ports map**

In the same file, replace the `to_rego_data_json` method body (keep the doc-comment intent, update it for the new shape):

```rust
    /// The regorus data document for `sandbox`: the allow-list becomes this
    /// sandbox's `sandbox_ports[<name>]` host→ports map (`global_domains` stays
    /// empty — a `--policy` file is scoped to one sandbox, never granted to
    /// others). A bare host maps to the default web ports; a scoped host to its
    /// exact set.
    pub fn to_rego_data_json(&self, sandbox: &str) -> String {
        let mut ports = serde_json::Map::new();
        for entry in &self.allow {
            ports.insert(entry.host().to_string(), serde_json::json!(entry.ports()));
        }
        serde_json::json!({
            "global_domains": {},
            "sandbox_ports": { sandbox: ports },
        })
        .to_string()
    }
```
